## User Story

**As a** Developer  
**I want** a use case manifest schema (YAML/JSON) that declaratively defines name, description, category, parameters, file requirements, plugin endpoint, timeout and output definitions  
**so that** any business use case can be described declaratively and the platform can auto-generate UI and dispatch execution from the schema

| Field | Value |
|-------|-------|
| **BRD §** | §22 — Process-First Architecture Pivot, §22.4 — Use Case Manifest Schema, §22.6 — Hybrid Component Architecture |
| **Business Value** | Enables zero-code onboarding of new business use cases — define a manifest and the platform handles UI, validation and dispatch |
| **Current State** | No standardized format for describing business use cases or their requirements |
| **Problem** | Without a schema, each use case needs custom UI, validation and dispatch logic |
| **Solution** | Define a manifest schema with typed parameters, file requirements, plugin endpoints and optional custom UI configuration; validate with JSON Schema / Pydantic |

## Technical Specification

### Pydantic Schema Models

```python
# services/api-gateway/app/domain/models/manifest.py
from __future__ import annotations
from enum import Enum
from typing import Any
from pydantic import BaseModel, Field, model_validator


class ParameterType(str, Enum):
    STRING = "string"
    NUMBER = "number"
    INTEGER = "integer"
    BOOLEAN = "boolean"
    DATE = "date"
    DATETIME = "datetime"
    STRING_ARRAY = "string[]"
    NUMBER_ARRAY = "number[]"
    SELECT = "select"


class OutputType(str, Enum):
    FILE = "file"
    FILE_ARCHIVE = "file_archive"
    JSON = "json"
    TEXT = "text"


class ManifestParameter(BaseModel):
    """A typed parameter definition for a use case."""
    name: str = Field(..., min_length=1, max_length=100, pattern=r"^[a-z][a-z0-9_]*$")
    type: ParameterType
    label: str = Field(..., min_length=1, max_length=255)
    required: bool = False
    default: Any = None
    options: list[str] | None = None  # Required for 'select' type
    description: str | None = None

    @model_validator(mode="after")
    def validate_select_options(self) -> ManifestParameter:
        if self.type == ParameterType.SELECT and (not self.options or len(self.options) == 0):
            raise ValueError(f"Parameter '{self.name}' of type 'select' must have non-empty 'options'")
        return self


class ManifestFile(BaseModel):
    """A file upload requirement for a use case."""
    name: str = Field(..., min_length=1, max_length=100, pattern=r"^[a-z][a-z0-9_]*$")
    label: str = Field(..., min_length=1, max_length=255)
    accepted_types: list[str] = Field(..., min_length=1)
    max_size_mb: float = Field(default=100.0, gt=0, le=1000)
    required: bool = True
    description: str | None = None


class ManifestPlugin(BaseModel):
    """Plugin endpoint configuration."""
    endpoint: str = Field(..., min_length=1)
    health_check: str = Field(default="/health")
    timeout_seconds: int = Field(default=300, ge=10, le=3600)


class ManifestOutput(BaseModel):
    """An output definition for a use case."""
    name: str = Field(..., min_length=1, max_length=100)
    type: OutputType
    description: str | None = None
    mime_type: str | None = None


class ManifestUIType(str, Enum):
    """How the platform renders the use case view."""
    AUTO = "auto"        # Default: auto-generate form from parameters/files
    REMOTE = "remote"    # Load custom React component via Module Federation


class ManifestUI(BaseModel):
    """
    Optional UI override section (BRD §22.6).
    
    When absent or type="auto", the platform auto-generates the use case form
    from parameters and files. When type="remote", the platform loads a
    plugin-contributed React component at runtime via Webpack Module Federation.
    
    The loaded component receives UseCaseViewProps from the parent shell.
    """
    type: ManifestUIType = Field(default=ManifestUIType.AUTO)
    component_url: str | None = Field(
        default=None,
        description="URL to Webpack Module Federation remoteEntry.js"
    )
    component_name: str | None = Field(
        default=None,
        description="Named export of the React component from the remote module"
    )
    fallback: str = Field(
        default="auto",
        pattern=r"^(auto|error)$",
        description="Behaviour on load failure: 'auto' = render auto-gen form, 'error' = show error"
    )
    version: str = Field(
        default="1.0.0",
        pattern=r"^\d+\.\d+\.\d+$",
        description="Cache-busting version appended to component_url"
    )

    @model_validator(mode="after")
    def validate_remote_fields(self) -> ManifestUI:
        if self.type == ManifestUIType.REMOTE:
            if not self.component_url:
                raise ValueError("ui.component_url is required when ui.type = 'remote'")
            if not self.component_name:
                raise ValueError("ui.component_name is required when ui.type = 'remote'")
        return self


class UseCaseManifest(BaseModel):
    """
    Complete use case manifest schema.
    
    Defines everything needed for the platform to:
    1. Auto-generate the parameter form UI (E20-S03) OR load a custom component (BRD §22.6)
    2. Validate user inputs (parameters and files)
    3. Dispatch execution to the plugin (E22-S02)
    4. Present results to the user (E20-S06)
    """
    name: str = Field(..., min_length=1, max_length=100, pattern=r"^[a-z][a-z0-9-]*$")
    display_name: str = Field(..., min_length=1, max_length=255)
    description: str = Field(..., min_length=1)
    category: str = Field(..., min_length=1, max_length=100)
    icon: str | None = None
    version: str = Field(default="1.0.0", pattern=r"^\d+\.\d+\.\d+$")
    
    parameters: list[ManifestParameter] = Field(default_factory=list)
    files: list[ManifestFile] = Field(default_factory=list)
    plugin: ManifestPlugin
    outputs: list[ManifestOutput] = Field(default_factory=list)
    
    # Optional: custom UI override (BRD §22.6 Hybrid Component Architecture)
    # When absent or ui.type="auto", the platform auto-generates the form.
    # When ui.type="remote", a React component is loaded from ui.component_url.
    ui: ManifestUI | None = Field(
        default=None,
        description="Optional custom UI configuration. None = auto-generate from parameters/files."
    )

    @model_validator(mode="after")
    def validate_unique_names(self) -> UseCaseManifest:
        param_names = [p.name for p in self.parameters]
        if len(param_names) != len(set(param_names)):
            raise ValueError("Parameter names must be unique within a manifest")
        file_names = [f.name for f in self.files]
        if len(file_names) != len(set(file_names)):
            raise ValueError("File names must be unique within a manifest")
        return self
```

### YAML/JSON Parser Utility

```python
# services/api-gateway/app/domain/services/manifest_parser.py
import json
import yaml
from pathlib import Path
from app.domain.models.manifest import UseCaseManifest


def parse_manifest(content: str, format: str = "yaml") -> UseCaseManifest:
    """
    Parse a use case manifest from YAML or JSON string.
    
    Args:
        content: Raw manifest content (YAML or JSON string)
        format: "yaml" or "json"
    
    Returns:
        Validated UseCaseManifest instance
    
    Raises:
        ValueError: If content is invalid YAML/JSON
        ValidationError: If manifest fails schema validation
    """
    if format == "json":
        data = json.loads(content)
    else:
        data = yaml.safe_load(content)
    
    return UseCaseManifest(**data)


def parse_manifest_file(path: Path) -> UseCaseManifest:
    """Parse a manifest from a file, auto-detecting format from extension."""
    content = path.read_text()
    fmt = "json" if path.suffix in (".json",) else "yaml"
    return parse_manifest(content, format=fmt)
```

## Business Logic

### Rules

| # | Rule | Enforcement |
|---|------|-------------|
| 1 | `name` must be lowercase kebab-case | Pydantic regex pattern |
| 2 | Parameter `name` must be lowercase snake_case | Pydantic regex pattern |
| 3 | `select` type parameters must have non-empty `options` | Pydantic model_validator |
| 4 | Parameter and file names must be unique within a manifest | Pydantic model_validator |
| 5 | Plugin endpoint is required | Pydantic required field |
| 6 | `max_size_mb` must be > 0 and ≤ 1000 | Pydantic Field constraints |
| 7 | `timeout_seconds` must be between 10 and 3600 | Pydantic Field constraints |
| 8 | Version must be semantic versioning format | Pydantic regex pattern |
| 9 | `ui.component_url` and `ui.component_name` are required when `ui.type = "remote"` | Pydantic model_validator |
| 10 | `ui.fallback` must be `"auto"` or `"error"` | Pydantic regex pattern |
| 11 | `ui` section is optional; when absent, defaults to `None` (auto-generated UI) | Pydantic Field default |

### Supported Parameter Types

| Type | Python Mapping | UI Component (E20-S03) | Description |
|------|---------------|----------------------|-------------|
| `string` | `str` | Text input | Free text |
| `number` | `float` | Number input | Decimal number |
| `integer` | `int` | Number input (step=1) | Whole number |
| `boolean` | `bool` | Checkbox/Toggle | True/false |
| `date` | `date` | Date picker | Date only |
| `datetime` | `datetime` | DateTime picker | Date and time |
| `string[]` | `list[str]` | Multi-value text input | Multiple strings |
| `number[]` | `list[float]` | Multi-value number input | Multiple numbers |
| `select` | `str` (from options) | Select dropdown | One of predefined options |

| Case | Behaviour | Test |
|------|-----------|------|

## Notes

- This schema is the **foundation** for E21-S02 (CRUD API), E20-S03 (UI generation), E20-S02 (conditional remote component loader) and E22-S02 (execution dispatch)
- The manifest is stored as-is in `use_cases.manifest_json` column (T00-02 schema)
- Individual parameters and files are also stored in normalized tables for query efficiency
- The optional `ui` section (BRD §22.6) enables plugins to ship custom React components loaded via Webpack Module Federation. When `ui` is absent, the platform auto-generates the form — this is the default, zero-effort path.
- Phase 2 will extend with MCP-compatible tool manifest fields (from original E12)
